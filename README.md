# ระบบจัดการสินค้าลองตัวอย่าง (Sample Part)

---

## 1. ระบบเบิกสินค้าลองตัวอย่าง (Sample Part Issue)

### การมองเห็น Item ตามหน่วยงาน

| หน่วยงาน | รหัส Item ที่มองเห็น |
|---|---|
| **Warehouse** | 3B, 4B, 5, 6 |
| **Store** | 1A, 1S, 1K, 1W, 1R, 1C, 1B, 2R, 2C, 1PACK, 1FOAM, 1FOIL |

> **Stored Procedure สำหรับ Filter Item:** `Rpt_SamplePartsIssueItem`

### การระบุเหตุผลการเบิก
- ต้องระบุเหตุผลการเบิกทุกครั้ง
- บันทึกข้อมูลที่ตาราง `tbReasonSampleParts` โดยกำหนด `Transtype = ISSUE`

### ขั้นตอนการทำงาน

```
สร้างเอกสาร → กดส่งอนุมัติ → ระบบ Move ยอดไปที่ Location: WAITISSUE
→ อนุมัติเรียบร้อย → ฝ่าย Store ทำการตัดจ่าย
```

### ลำดับการส่งอนุมัติ

| หน่วยงาน | ลำดับการอนุมัติ |
|---|---|
| **AutoEv** | คุณกัญญาพัชร เหมะรักษ์ *(เจาะจงผู้อนุมัติ)* |
| **Packaging Samut Sakhon** | Supervisor ผู้ขอ → Manager ผู้ขอ → คุณเนตรนภา บัวกลีบ *(เจาะจง)* → คุณอรทัย คุ้มแผน *(เจาะจง)* |
| **Packaging Chonburi** | Supervisor ผู้ขอ → Manager ผู้ขอ |

## 2. ระบบรับสินค้าลองตัวอย่าง (Sample Part Receive)

### การมองเห็น Item ตามหน่วยงาน

| หน่วยงาน | รหัส Item ที่มองเห็น |
|---|---|
| **Warehouse** | 3, 4, 5 |
| **Store** | 1, 8, 2C, 2R |

> **Stored Procedure สำหรับ Filter Item:** `spSamplePartsReceiptItem`

### การระบุเหตุผลการรับ
- ต้องระบุเหตุผลการรับทุกครั้ง
- บันทึกข้อมูลที่ตาราง `tbReasonSampleParts` โดยกำหนด `Transtype = RECEIVED`

### ขั้นตอนการทำงาน

```
สร้างเอกสาร → ส่งอนุมัติ → อนุมัติเรียบร้อย → ฝ่าย Store ทำการรับเข้า
```

### ลำดับการส่งอนุมัติ (เจาะจงผู้อนุมัติ)

| หน่วยงาน | ผู้อนุมัติ |
|---|---|
| **AutoEv** | คุณธรรศพงศ์ ภูวราลภัสธรณ์ |
| **Packaging Chonburi** | คุณทัดดาว เทียมพิทักษ์ |
| **Packaging Samut Sakhon** | คุณเนตรนภา บัวกลีบ |

---

## 3. เงื่อนไขร่วม (Issue & Receive)

### 3.1 การเพิ่มเหตุผลการเบิก/รับ
- หากผู้ใช้ต้องการเพิ่มเหตุผล **ต้องแจ้งฝ่ายบัญชีให้สร้างในระบบ Syteline ก่อน**
- เมื่อสร้างใน Syteline แล้วจึงจะสามารถเพิ่มใน `tbReasonSampleParts` ได้
- **ห้ามเพิ่มใน `tbReasonSampleParts` โดยตรงหากยังไม่ได้สร้างใน Syteline โดยเด็ดขาด**

### 3.2 การยกเลิกเอกสารอัตโนมัติ
- เอกสารที่อยู่ในสถานะ **รออนุมัติ** หรือ **อนุมัติแล้ว** เกิน **2 วันทำการ** ระบบจะยกเลิกให้อัตโนมัติ
- Stored Procedure: `spSamplePartsIssuePost_New`
- ตั้ง Schedule รันทุก **5 นาที**

---

## 4. ระบบรับและจ่ายสินค้าลองตัวอย่าง (Sample Part Management)

### วัตถุประสงค์
ใช้สำหรับรับและจ่ายสินค้าลองตัวอย่างจากทั้ง 2 ระบบข้างต้น (Issue & Receive)

### เงื่อนไขการมองเห็นเอกสาร
- แสดงเฉพาะเอกสารที่มีสถานะ **อนุมัติแล้ว** เท่านั้น

### Stored Procedure
| การทำงาน | Stored Procedure |
|---|---|
| ตัดจ่าย / รับเข้า | `spSamplePartsManagementPost` |
